# Controlled-path-Dual-sink-data-gathering

Chen et al. (2010) have tried to overcome data collisions and rapid energy consumption for some specific sensors which is caused by frequent location updates of the mobile sink. The mobile sink moves towards the event regions one after another. During its movement, the static sink collects all data packets from the source nodes. When the mobile sink reaches an event region, it stops until the event vanishes while it gathers a part of sensed data. The source nodes send the packets to the sinks in such a way that the energy consumption of the sinks is balanced equally. There are some parameters such as sensor transmission range (r) and the velocity of the mobile sink (v) for evaluating the energy performance of static sensors, Static Sink (SS) and Mobile Sink (MS). Nevertheless, this work focuses on energy balancing in SS and MS as it supposes that the sinks are power constrained. (Our implementation may a bit differ from the original work in details)


Chen, J., Salim, M.B. and Matsumoto, M. (2010). Modeling the Energy Performance of Event-Driven Wireless Sensor Network by Using Static Sink and Mobile Sink. Sensors. 10(12), 10876-10895.
